nixarch

Declarative Arch/CachyOS workstations, managed the Nix way.

A reusable base for running Arch-family desktops and laptops under Nix control without switching to NixOS: system-manager for the system layer, home-manager for the user layer, plus patterns for CachyOS-specific concerns expressed as data.

Status: pre-alpha

✓

Two real modules have landed

device-gids and gshadow-sync are real, working system-manager modules with documented options and usage — see the modules below. They are not a demo; the pattern behind them runs daily on a real Arch/CachyOS machine today.

!

Everything else is still not built

No home-manager base module, no CachyOS-specific data patterns, no worked end-to-end example machine configuration yet. This repository is being extracted from a private configuration one module at a time, not delivered as a finished tool.

!

Not a drop-in base yet

You can import and use the two landed modules on their own today (see Usage in the README). What you cannot do yet is point a whole Arch/CachyOS machine at nixarch and get a complete declarative base — that's the rest of the roadmap below.

What actually works today

device-gids

Pins shared device groups (render, video, and similar) to a caller-supplied gid map, and keeps them there. A oneshot groupmod migrates any pre-existing group to the requested gid — declaring a gid alone isn't enough, since userborn only creates missing groups at the declared gid and never renumbers an existing one. Including tty in the map also enables the devpts remount lockstep Arch needs to make the live pseudo-terminal gid track the group you chose.

gshadow-sync

Heals /etc/gshadow after userborn writes /etc/group: on Arch, system-manager realises groups through userborn, which never touches /etc/gshadow, so every group it creates leaves the daily shadow.service hygiene check permanently red. This module runs an idempotent grpconv that only touches the file when the two have actually diverged, and carries over to NixOS unchanged for the same reason.

system-manager on Arch, for real

system-manager is young, and almost nobody has published a serious, real-world configuration built on it for an Arch-family distro. The two modules above are the start of a worked, opinionated base instead of a toy demo — more will land the same way, extracted from daily use rather than written speculatively.

Roadmap (planned, not yet built)

More system-manager modules

Additional modules beyond device-gids and gshadow-sync, extracted from the same real Arch/CachyOS machine as they prove themselves in daily use.

home-manager base module

A user-layer module extracted from real daily-driver dotfiles and packages, generalized away from any one machine's specifics.

CachyOS data patterns

A documented, data-driven way to express v3/znver repository selection and kernel/scheduler choice, instead of one-off shell scripts.

None of the above exists in this repository yet. This section will be replaced with real install instructions once there is something to install.

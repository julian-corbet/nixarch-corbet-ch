nixarch

Declarative Arch/CachyOS workstations, managed the Nix way.

A reusable base for running Arch-family desktops and laptops under Nix control without switching to NixOS: system-manager for the system layer, home-manager for the user layer, plus patterns for CachyOS-specific concerns expressed as data.

Status: pre-alpha

!

Not yet consumable

This repository is a scaffold. It is being extracted from a private, working fleet configuration, one module at a time. There is no installable release, no module set, and no documented usage yet.

!

Real usage, no packaging

The pattern behind nixarch runs daily on real Arch/CachyOS machines today — it just isn't public or generalized yet. That extraction work is what this repository tracks.

!

Flake skeleton only

flake.nix currently exports empty module attrsets with a doc comment explaining they're placeholders. It evaluates cleanly; it does not do anything yet.

Why this exists

system-manager on Arch, for real

system-manager is young, and almost nobody has published a serious, real-world configuration built on it for an Arch-family distro. nixarch aims to be a worked, opinionated example instead of a toy demo.

Nix without NixOS

The goal is declarative system services and files plus a declarative user environment, layered on top of a distro that keeps its own package manager and its own kernel/driver story — not a NixOS migration.

CachyOS concerns as data

v3/znver-optimized repositories, kernel and scheduler choices, and other CachyOS-specific decisions are meant to be expressed as configuration data consumed by the modules, not hardcoded into them.

Roadmap (planned, not yet built)

system-manager base module

A minimal, documented system-manager configuration covering system services and files that are safe to apply on an unmodified Arch/CachyOS install.

home-manager base module

A user-layer module extracted from real daily-driver dotfiles and packages, generalized away from any one machine's specifics.

CachyOS data patterns

A documented, data-driven way to express v3/znver repository selection and kernel/scheduler choice, instead of one-off shell scripts.

None of the above exists in this repository yet. This section will be replaced with real install instructions once there is something to install.
